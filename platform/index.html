How it works

Point it at a target. Get back working exploits.

Kryx runs the entire pentest autonomously — from the context you give it to a confirmed, working exploit, every time your apps change.

  1. 01

    Recon

    Kryx maps your attack surface: domains, subdomains, open ports, and the full stack behind them — a live picture before the first probe.

  2. 02

    Hunt

    Eight specialized agents attack in parallel — injection, XSS, auth, config, CORS, network, secrets, CVEs — each driving the right tool.

  3. 03

    Validate

    An adversarial agent tries to disprove every finding. Only what survives reaches you. Zero false positives.

  4. 04

    Chain

    Proven findings link into APT-grade kill chains — paths a real attacker would walk that scanners never reach.

  5. 05

    Report

    A board-ready report drops the moment the run ends: chain, working exploit, and developer-ready remediation.

Breaking a Cryptographic Captcha — CBC Padding Oracle

CRITICALCVSS 9.8CWE-326OPEN

The captcha cookie is a raw AES-CBC token with no MAC, so its padding check doubles as a decryption oracle. The agent recovered the plaintext byte-by-byte, forged a valid token, and registered an account with no human ever solving the captcha.

  1. 01Fingerprinted the captcha cookie as a 256-bit AES-CBC token.
  2. 02Found the verify endpoint leaks padding validity in its response.
  3. 03Built a CBC padding oracle against that endpoint.
  4. 04Decrypted the cookie byte-by-byte to recover its structure.
  5. 05Forged a valid captcha token from the recovered format.
  6. 06Registered a user and captured the flag — zero human input.
